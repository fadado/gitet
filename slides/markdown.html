Gitet

Què és Gitet?

  • Gitet és un petitet primer contacte amb Git i el seu ecosistema
  • Gitet consta de dues presentacions:
    1. Git i el seu ecosistema
    2. Markdown
  • Gitet està disponible en un repositori de GitHub i un de GitLab
  • Gitet està escrit amb Markdown i generat amb Pandoc
  • Gitet és un exemple de la filosofia Eating your own dog food

Què és Markdown?

  • Markdown és un llenguatge de marques extremadament simple,
  • que permet escriure utilitzant simple text per després convertir-lo en altres formats.
  • Markdown va ser creat originalment per John Gruber i Aaron Swartz (película)
  • prenent inspiració en les convencions tradicionals a l’hora de formatar text en correus electrònics.
  • Markdown no és més que simple text amb certes convencions!

Alternatives a MarkDown

Aquestes alternatives són molt més complexes i potents que MarkDown.

  • reStructuredText (documentació de Python)
  • AsciiDoc (documentació del Kernel de Linux)

Software a usar

A executar en un terminal com root:

yum -y install geany geany-plugins-markdown
yum -y install pandoc discount
yum -y install aspell aspell-ca

També és practic usar Google Chrome amb la extensió MarkDown Preview Plus o equivalent.

Podem corregit l’ortografia dels documents fets amb Markdown amb ordres com aquesta:

aspell --lang=ca check README.md

Què farem?

  • Crear un repositori local de Git pel nostre projecte
  • Editar Markdown amb Geany
  • Convertir Markdown a HTML, ODT, PDF, &c. amb Pandoc
  • Visualitzar Markdown una vegada pujat a GitHub o GitLab
  • Editar Markdown amb http://dillinger.io/
  • Modificar Gitet i pujar els canvis a GitHub
  • &c.
